# 프로젝트 개요

## 프로젝트명
**LLM Selector** — 실시간 데이터 기반 AI 모델 선택 어드바이저

## 한 줄 요약
사용자의 용도를 분석하고, 최신 모델 데이터를 기반으로 합리적인 모델 선택을 도와주는 웹 애플리케이션

---

## 문제 정의

LLM 모델 시장은 새로운 모델이 빠르게 출시되고, 기존 모델의 가격과 성능도 수시로 변동한다. 모델 발전의 기본 추세는 성능 향상 + 가격 하락이며, 이 속도는 점점 빨라지고 있다.

### 핵심 문제
1. **정보 컷오프**: 기존 AI 챗봇에게 물어보면 학습 데이터 컷오프 이전의 정보를 기반으로 답변하여, 이미 더 좋고 더 저렴한 모델이 나왔음에도 오래된 모델을 추천
2. **비용 영향**: 모델 선택 하나가 API 기반 서비스의 월 비용을 몇 배씩 차이 나게 만들 수 있음
3. **비교 어려움**: 벤치마크 숫자만으로는 "이게 내 상황에서 어떤 의미인지" 판단하기 어렵고, 정보가 여러 곳에 흩어져 있음
4. **무비판적 선택**: 특정 모델을 도입할 때 일반적으로 유명한 모델만을 디테일한 검토 없이 의사결정하는 패턴

---

## 해결 방식

### 설계 원칙: LLM이 잘하는 것과 못하는 것의 분리

| LLM이 잘하는 것 | LLM이 못하는 것 |
|----------------|----------------|
| 자연어 이해, 분석, 구조화 | 최신 정보 제공 |
| 용도 분석, 비교 추론 | 정확한 가격/성능 데이터 |

LLM은 모델 정보를 "알고 있는 것"으로 답하지 않는다. 대신 **에이전트 워크플로우**를 통해:
1. 사용자의 자연어 용도를 분석하는 **두뇌 역할**
2. 도구(Tools)를 **자율적으로** 활용하여 모델 DB 조회, 벤치마크 비교, 웹 검색 수행
3. 투명한 근거와 함께 추천 제공

### 데이터베이스 전략
- OpenRouter API를 통해 400+ 모델의 가격, 컨텍스트 길이, 기능 등 자동 동기화
- 벤치마크, 모델 특성 태그 등은 수동/별도 소스에서 보강 (하이브리드)
- 매 요청마다 크롤링이 아니라 사전 구축된 DB에서 조회 → 빠른 응답 속도

---

## 핵심 가치

**정보의 신선도** — 항상 최신 모델 데이터를 반영하여, 사용자가 시점에 관계없이 현재 최적의 선택을 할 수 있도록 한다.

---

## 타겟 사용자

모델 도입 시 디테일한 검토 없이 유명한 모델만 선택하는 **모든 의사결정자**:
- AI 애플리케이션을 개발하는 개발자
- 어떤 모델을 도입할지 결정해야 하는 PM, 기획자, 경영진

---

## MVP 범위 (2주)

노션 원문의 서비스 구성 중 **1~3단계**를 MVP로 구현:
1. **용도 분석**: 자연어 → 구조화된 요구사항
2. **모델 매칭**: DB 기반 후보 모델 선정
3. **근거 기반 비교 리포트**: 구조화된 비교표 + 추천 이유

4단계(실전 비교 테스트)는 MVP 이후 확장 기능으로 진행.
