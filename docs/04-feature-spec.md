# 기능 명세 (MVP)

## F1. API 키 입력 및 관리

### 설명
사용자가 OpenRouter API 키를 입력하여 세션 동안 사용. 이 키로 분석용 LLM 호출과 향후 실전 비교 테스트를 모두 처리.

### 상세
- **저장**: 브라우저 sessionStorage (서버에 저장하지 않음 — 보안)
- **검증**: OpenRouter `GET /api/v1/key` 엔드포인트로 유효성 확인
- **표시 정보**: 키 유효 여부, 잔여 크레딧, 사용량 (일/주/월)
- **UI 위치**: 상단 고정 바

### UI 동작
1. 키 미입력 상태: "OpenRouter API 키를 입력하세요" + 입력 필드
2. 키 입력 후: 검증 중 로딩 → 성공 시 "✓ 유효 | 잔여: $12.50" 표시
3. 키 무효: 에러 메시지 + 재입력 유도
4. 키 변경: 기존 키 마스킹 표시 + "변경" 버튼

## F2. 에이전트 기반 모델 추천

### 설명
노션 원문의 1~3단계를 에이전트 워크플로우로 통합. 사용자가 자연어로 용도를 설명하면, ModelAdvisorAgent가 자율적으로 도구를 호출하여 분석→매칭→리포트를 생성.

### 입력
- 자연어 텍스트 (e.g., "한국어 고객 상담 챗봇을 만들고 싶어")
- 분석용 LLM 모델 선택 (드롭다운, 기본값: 비용효율적인 모델 추천)

### 처리
FastAPI 백엔드의 ModelAdvisorAgent가:
1. 용도를 분석하여 요구사항 구조화
2. 자율적으로 도구 호출 (search_models, compare_pricing, get_benchmarks, web_search, assess_model_fit)
3. 결과를 종합하여 ComparisonReport 생성

### 스트리밍 (SSE)
에이전트의 추론 과정을 실시간으로 프론트엔드에 전달:
- `thinking`: 에이전트 사고 과정
- `tool_call`: 어떤 도구를 왜 호출하는지
- `tool_result`: 도구 실행 결과 요약
- `report`: 최종 비교 리포트
- `done`: 완료

### 출력
ComparisonReport (→ 02-agent-workflow.md 참조)

### 투명성
- **기본**: 결과(비교 대시보드)만 표시
- **"추론 과정 보기" 토글**: 에이전트가 어떤 도구를 왜 호출했고, 어떤 결과를 받았는지 단계별로 표시

### UI 동작
1. 용도 입력 폼 작성 + "분석 시작" 클릭
2. SSE 연결 시작 → 에이전트 추론 과정 스트리밍 (로딩 + 진행 표시)
3. 완료 시 비교 대시보드 표시

## F3. 비교 대시보드

### 설명
에이전트가 생성한 ComparisonReport를 시각적으로 표현하는 대시보드.

### 구성 요소

#### Top Pick 카드
- 최우선 추천 모델 + 추천 이유
- 강조 표시 (배지, 색상 구분)

#### 비교표
| 항목 | 내용 |
|------|------|
| 모델명 + 제공사 | 아이콘 포함 |
| 종합 점수 | 0-100, 프로그레스 바 |
| 가격 점수 | 0-100 |
| 성능 점수 | 0-100 |
| 적합도 점수 | 0-100 |
| 월 예상 비용 | $xxx ~ $xxx |
| 최적 용도 | "이런 경우에 최적" |

#### 상세 카드 (각 모델)
- 강점 목록 (이 용도에서의 강점)
- 약점 목록 (이 용도에서의 약점)
- 월 비용 시뮬레이션 (저/중/고 사용량)
- 벤치마크 주요 점수

#### 요약
- 전체 비교를 자연어로 요약
- 근거 출처 목록 (어떤 데이터를 기반으로 판단했는지)

## F4. 모델 탐색

### 설명
전체 모델 DB를 브라우징할 수 있는 탐색 페이지. 에이전트 추천과 별도로, 사용자가 직접 모델을 검색하고 비교할 수 있음.

### 기능
- **검색**: 모델명, 제공사, 설명으로 텍스트 검색
- **필터**: 제공사별, 가격대별, 컨텍스트 길이별, 기능별 (vision, tools 등), 무료/유료
- **정렬**: 가격 오름/내림, 컨텍스트 길이, 이름
- **페이지네이션**: 무한 스크롤 또는 페이지 번호

### 모델 상세 페이지 (/models/[id])
- 전체 프로필: 가격, 컨텍스트 길이, 지원 기능, 아키텍처
- 벤치마크 점수표 (있는 경우)
- 특성 태그
- OpenRouter 페이지 링크

## F5. 모델 DB 동기화

### 설명
OpenRouter API에서 모델 데이터를 자동으로 동기화하여 DB를 최신 상태로 유지.

### 자동 동기화
- **주기**: Render Cron Job, 매일 1회 (UTC 00:00 등)
- **소스**: OpenRouter `GET /api/v1/models`
- **동작**:
  1. 전체 모델 목록 조회
  2. 신규 모델: INSERT
  3. 기존 모델 정보 변경 (가격 등): UPDATE
  4. API에서 사라진 모델: is_active = false 처리

### 수동 트리거
- `/api/v1/sync` 엔드포인트로 즉시 동기화 실행 가능
- MVP에서는 간단한 관리 UI 또는 API 직접 호출

### 수동 보강
- 벤치마크 데이터 (model_benchmarks): 관리자가 입력
- 모델 특성 태그 (model_tags): 관리자가 입력
- 향후 자동화 검토 (벤치마크 사이트 크롤링 등)
