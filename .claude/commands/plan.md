# /plan — 기능 계획 수립

새 기능이나 복잡한 변경 전에 체계적인 구현 계획을 수립합니다.

## 실행 절차

1. **기획 문서 확인**: `docs/` 디렉토리에서 관련 문서 읽기
2. **코드베이스 탐색**: 영향 받는 파일과 패턴 파악
3. **planner 에이전트 호출**: 구현 계획 생성
4. **계획 문서 저장**: `docs/plans/PLAN_<feature>.md` 파일 생성
5. **사용자 승인 대기**: 승인 전까지 코드 작성 금지

## 계획 문서 경로

```
docs/plans/PLAN_<feature_name>.md
```

## 사용자 승인 후

승인되면 Phase별로 구현을 진행합니다.
각 Phase 완료 시 검증 실행 (테스트 + 빌드).
