# 버그 수정 워크플로우

## Phase 1: 근본 원인 추적

수정 전에 반드시 전체 실행 경로를 추적합니다.

1. **증상 파악** — 사용자가 보는 실패 동작 정리
2. **실행 경로 추적**
   - Frontend: 사용자 액션 → 컴포넌트 → API 호출
   - Backend: API 라우트 → 서비스 → 에이전트/도구 → DB
3. **분기점 식별** — 조건문, try/except, 리다이렉트 지점
4. **근본 원인 확정** — 동작이 예상과 갈라지는 정확한 지점

**출력 형식**:
```
증상: [사용자가 보는 것]
실행 경로: [A → B → C → ❌ D (여기서 갈라짐)]
근본 원인: [왜 D에서 실패하는지]
```

## Phase 2: 수정 구현

1. **최소 범위 수정** — 근본 원인에 가장 가까운 레이어에서 수정
2. **에러 처리** — except 후에도 정상 응답이 완료되는지 확인
3. **관련 파일 모두 수정** — 모델, 라우트, 서비스, 테스트 동시

### 3회 실패 시 중단 규칙

같은 버그에 3회 이상 실패하면:

1. **즉시 중단**
2. **시도 정리**:
   | 시도 | 가설 | 변경 | 실패 이유 |
   |------|------|------|----------|
3. **아키텍처 재고** — 설계 전제가 잘못되었는지 탐색
4. **사용자 보고** — 상황 정리 + 대안 2-3개 제시

## Phase 3: 검증

```bash
# Backend
cd backend && pytest

# Frontend
cd frontend && npm run build && npm run test

# 타입 체크
cd frontend && npx tsc --noEmit
```

1. 전체 테스트 통과 확인
2. 커밋 메시지: `fix: <근본 원인과 수정 설명>`

## 체크리스트

```

- [ ] 근본 원인 확정 (증상이 아닌 원인)
- [ ] 수정 범위 최소화 확인
- [ ] 전체 테스트 통과
```
