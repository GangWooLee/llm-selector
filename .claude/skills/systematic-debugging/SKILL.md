# 체계적 디버깅 스킬

> **📚 관련 기획 문서**: `docs/02-agent-workflow.md` (에이전트 흐름), `docs/05-api-spec.md` (API 스펙)

기존 `bugfix` 스킬보다 강화된 5단계 근본 원인 분석 워크플로우.
SSE 스트리밍 + Pydantic AI 에이전트 + OpenRouter API 조합의 복합 버그에 특화.

## 이 프로젝트의 실행 경로

```
사용자 브라우저
  → Next.js SSE Client (fetch + ReadableStream)
    → FastAPI Route (/api/v1/advise)
      → AdvisorService
        → ModelAdvisorAgent (Pydantic AI)
          → Tool: search_models → DB (queries.py)
          → Tool: compare_pricing → DB
          → Tool: get_benchmarks → DB
          → Tool: assess_model_fit → DB + 계산
          → Tool: web_search → Tavily API (외부)
          → Tool: get_model_details → DB
        → ComparisonReport 생성
      → SSE 이벤트 변환
    → StreamingResponse
  → 프론트엔드 파싱 + 렌더링
```

## 5단계 프로세스

### Phase 1: 근본 원인 조사 (코드 수정 금지)

1. **에러 메시지 정확히 읽기** — 추측하지 말고 실제 에러 전문 확인
2. **재현 시도** — 동일 입력으로 재현 가능한지 확인
3. **최근 변경 확인** — `git diff HEAD~3` 로 최근 변경이 원인인지
4. **경계 지점 증거 수집** — 각 레이어 경계에서 실제 값 확인:
   - SSE Client → FastAPI: 요청 바디, 헤더
   - Route → Service: params 전달값
   - Service → Agent: 에이전트 실행 시작 여부
   - Agent → Tool: 도구 호출 파라미터, 반환값
   - Tool → DB: 쿼리 결과
5. **근본 원인 확정** — 동작이 예상과 갈라지는 **정확한 지점** 특정

**산출물**:
```
증상: [사용자가 보는 것]
경로: [A → B → C → ❌ D (여기서 갈라짐)]
경계 증거: [C→D 경계에서 실제 값: X, 예상 값: Y]
근본 원인: [왜 D에서 실패하는지]
```

### Phase 2: 패턴 분석

1. **동작하는 유사 코드 찾기** — 같은 패턴이 다른 곳에서 작동하는지
2. **차이점 비교** — 작동하는 것과 실패하는 것의 차이
3. **히스토리 확인** — 이전에 작동했다면 언제부터 깨졌는지

### Phase 3: 가설 수립 및 테스트

1. **단일 가설** 수립 (여러 가설 동시 테스트 금지)
2. **최소한의 변경**으로 가설 테스트
3. **결과 기록** — 성공/실패 + 이유

### Phase 4: 수정 구현

1. **근본 원인에 가장 가까운 레이어**에서 수정
2. **실패를 재현하는 테스트** 먼저 작성
3. **수정 적용** → 테스트 통과 확인
4. **전체 테스트** 실행

### Phase 5: 아키텍처 재고 (3회 실패 시)

3번 연속 수정이 실패하면:

1. **즉시 중단** — 추가 수정 시도 금지
2. **시도 정리**:

| 시도 | 가설 | 변경 | 실패 이유 |
|------|------|------|----------|
| 1 | ... | ... | ... |
| 2 | ... | ... | ... |
| 3 | ... | ... | ... |

3. **전제 재검토**:
   - 설계 전제가 잘못되었나?
   - 다른 레이어에서 해결해야 하나?
   - 접근법 자체를 바꿔야 하나?
4. **사용자 보고** — 상황 + 대안 2-3개 제시

## 프로젝트 고유 디버깅 패턴

### SSE 스트리밍 문제
| 증상 | 가능한 원인 | 확인 방법 |
|------|-----------|----------|
| 이벤트가 안 옴 | Nginx 버퍼링 | `X-Accel-Buffering: no` 헤더 확인 |
| 이벤트 순서 꼬임 | async generator 순서 | 백엔드 로그에서 yield 순서 확인 |
| 마지막 이벤트만 옴 | 버퍼링 후 일괄 전송 | `Cache-Control: no-cache` 확인 |
| 프론트에서 파싱 실패 | SSE 포맷 오류 | `\n\n` 이벤트 구분자 확인 |

### 에이전트 문제
| 증상 | 가능한 원인 | 확인 방법 |
|------|-----------|----------|
| 도구를 안 호출함 | 시스템 프롬프트 누락 | prompts.py 확인 |
| 잘못된 도구 호출 | docstring 불명확 | 도구 설명 개선 |
| 타임아웃 | 도구 내부 무한 루프/느린 쿼리 | 개별 도구 단독 테스트 |
| 자체 지식으로 답변 | 시스템 프롬프트 무시 | 프롬프트에 제약 강화 |

## 금지 사항

- ❌ 증상만 보고 바로 수정 (근본 원인 먼저)
- ❌ "빠른 수정" (quick fix)
- ❌ 여러 가설 동시 테스트
- ❌ 같은 수정 반복 시도
