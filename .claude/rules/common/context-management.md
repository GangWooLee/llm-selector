# 컨텍스트 관리 — 세션 품질 유지

## 컨텍스트 열화 신호

다음 증상이 나타나면 즉시 대응:
- 같은 세션에서 이전 지시를 잊어버림
- 이미 답한 질문을 반복함
- 프로젝트 규칙에 어긋나는 코드 생성
- 파일 경로나 함수명을 잘못 기억

## 세션 경계 판단

| 상황 | 조치 |
|------|------|
| 새 기능 (다른 도메인) | 새 세션 시작 |
| 같은 기능, 다음 Phase | Phase 경계에서 `/compact` |
| 디버깅 3회+ 실패 | 새 세션 시작 |
| 대규모 탐색 완료 → 구현 시작 | `/compact` 후 구현 |
| frontend ↔ backend 전환 | 해당 도메인 docs 다시 읽기 |

## 전략적 컴팩트 타이밍

### ✅ 좋은 타이밍 (이때 `/compact`)
- 대규모 탐색/조사 완료 후 → 구현 시작 전
- Phase 완료 후 → 다음 Phase 시작 전
- 디버깅 해결 후 → 다음 작업 전
- frontend ↔ backend 전환 시

### ❌ 나쁜 타이밍 (이때 하지 말 것)
- 구현 중간 (파일 수정 도중)
- 디버깅 중 (가설 추적 도중)
- 테스트 작성 중 (RED-GREEN 사이클 도중)
- 리팩토링 중 (변경 완료 전)

### 사전 체크리스트 (컴팩트 전 확인)
- [ ] 현재 작업이 논리적 단위로 완료됨
- [ ] 다음 단계가 새로운 컨텍스트를 필요로 함
- [ ] 핵심 결정과 발견이 파일(task_plan.md 등)에 기록됨
