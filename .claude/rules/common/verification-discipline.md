# 검증 규율 — 증거 없이 완료 선언 금지

## 원칙: Evidence before claims, always

## 6단계 검증 시퀀스

1. **명령어 식별** — 주장을 증명할 검증 명령어를 결정
2. **새로 실행** — 캐시나 기억이 아닌, 명령어를 지금 실행
3. **전체 출력 확인** — exit code, 실패 건수, 경고 모두 읽기
4. **일치 확인** — 출력이 주장과 일치하는지 판별
5. **증거 첨부** — 증거를 첨부한 후에만 완료 선언
6. **재검증** — 놓친 엣지 케이스, 다른 기능 영향 확인

## 즉시 중단해야 하는 위험 신호

| 위험 신호 | 올바른 행동 |
|----------|-----------|
| "아마 될 것입니다" | 명령어 실행으로 확인 |
| "완벽합니다!" (검증 전) | 검증 완료 후 판단 |
| 커밋 준비 (재실행 없이) | 테스트 + 린트 새로 실행 |

## 적용 시점

| 시점 | 프론트엔드 검증 | 백엔드 검증 |
|------|--------------|-----------|
| Phase 완료 시 | `npm run build && npm run test` | `pytest` |
| PR 전 | `/verify` (전체) | `/verify` (전체) |
| 커밋 전 | `npm run lint` + 관련 테스트 | `pytest` + 관련 테스트 |
| 버그 수정 후 | 재현 시나리오 재실행 | 재현 시나리오 재실행 |
